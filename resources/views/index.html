Path finding problem

Welcome to the presentation of the application created for finding a path on a small map with obstacles using genetic algorithm.

This application is created to demonstrate characteristics and challenges of using genetic algorithm for solving simple problem such as finding a path. Although there are several well known general algorithms, such as Dijkstra's Algorithm or A* Algorithm that are widely used for finding the optimal shortest path, here is used different approach - genetic algorithm.

Genetic Algorithms are adaptive heuristic search algorithm based on the evolutionary ideas of natural selection and genetics. As such they represent an intelligent exploitation of a random search used to solve optimization problems. In a genetic algorithm, a population of candidate solutions (called individuals, creatures, or phenotypes) to an optimization problem is evolved toward better solutions. Each candidate solution has a set of properties (its chromosomes or genotype) which can be mutated and altered. The evolution usually starts from a population of randomly generated individuals, and is an iterative process, with the population in each iteration called a generation. In each generation, the fitness of every individual in the population is evaluated. The more fit individuals are stochastically selected from the current population and each individual's genome is modified (recombined (using crossover operator) and possibly randomly mutated (using mutation operator)) to form a new generation. The new generation of candidate solutions is then used in the next iteration of the algorithm. Commonly, the algorithm terminates when either a maximum number of generations has been produced, or a satisfactory fitness level has been reached for the population.

Having said that, by using genetic algorithm for path finding problem several challenges had to be overcome. Firstly, fitness function used to determine the performance level of each individual had to take into consideration not only the distance from the goal on the map, but also the behaviour of the individual (how many times barriers or borders were hit etc.). Also, the way how individuals were selected for the crossover (two by two best individuals were selected to create a child for next generation) had great impact on the algorithm convergence. Mutation was performed on several genes in one genome and, at the end the new generation had several new random units that led to faster convergence.

You are now ready to run the application!
By clicking on the button, one path will be found and displayed on the map.

Please note that it may take some time for the algorithm to finish its search and that the path it is found may not always be optimal. This is to demonstrate the unpredictable behaviour of a genetic algorithm even in the circumstances when all parameters are set properly.
